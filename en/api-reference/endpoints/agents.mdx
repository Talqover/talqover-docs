# Voice Agents API

This document provides comprehensive documentation for the Voice Agents API endpoints, including request/response examples and validation rules.

## Base URL

```
/api/v1/agents
```

## Authentication

All endpoints require environment token authentication via the `EnvironmentTokenAuth` middleware.

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer talq_your_environment_token_here`
</ParamField>

### Create Agent

<RequestExample>

```bash
# Request 1: Create a basic inbound support agent

  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "label": "Customer Support Agent",
    "description": "Handles customer inquiries and support requests",
    "name": "SupportBot",
    "language": "en-US",
    "initial_message": "Hello, how can I help you today?",
```

## Enums Reference

### Agent Status
- `draft`: Agent is in draft mode
- `published`: Agent is published and active

### Agent Direction
- `inbound`: Agent receives incoming calls
- `outbound`: Agent makes outgoing calls

### Agent Language
- `en-US`: English (United States)
- `pt-BR`: Portuguese (Brazil)
- `es-ES`: Spanish (Spain)
- And other supported languages

### Interrupt Sensitivity
- `low`: Low sensitivity to interruptions
- `medium`: Medium sensitivity to interruptions
- `high`: High sensitivity to interruptions

### Who Speaks First
- `agent`: Agent speaks first
- `user`: User speaks first

### Active Status
- `active`: Item is active
- `inactive`: Item is inactive
