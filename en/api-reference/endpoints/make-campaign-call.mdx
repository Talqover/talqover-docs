# Make Campaign Call

Initiate a campaign call to multiple recipients with automated scheduling and tracking capabilities. This endpoint allows you to create outbound call campaigns with custom messaging and recipient management.

## Endpoint

```
POST /api/v1/campaigns/{campaign_id}/calls
```

## Path Parameters

<ParamField path="campaign" type="string" required>
  The unique identifier of the campaign. You can find this in your dashboard under Campaigns.
</ParamField>

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be set to `application/json`
</ParamField>

## Request Body

<ParamField body="name" type="string" required>
  Campaign name (max 255 characters)
</ParamField>

<ParamField body="description" type="string" required={false}>
  Campaign description
</ParamField>

<ParamField body="agent_id" type="string" required>
  The unique identifier of the voice agent to use for the campaign
</ParamField>

<ParamField body="recipients" type="array" required>
  Array of recipient phone numbers and metadata
</ParamField>

<ParamField body="schedule" type="object" required={false}>
  Campaign scheduling configuration
</ParamField>

<ParamField body="settings" type="object" required={false}>
  Campaign-specific settings and preferences
</ParamField>

## Error Codes

| Code | Description | HTTP Status |
|------|-------------|-------------|
| `AGENT_NOT_FOUND` | Specified agent does not exist | 404 |
| `INVALID_TOKEN` | Authentication token is invalid or missing | 401 |
| `UNAUTHORIZED` | User does not have permission to create campaigns | 403 |
| `VALIDATION_ERROR` | Request data validation failed | 422 |
| `RATE_LIMIT_EXCEEDED` | Too many campaigns created recently | 429 |
| `INSUFFICIENT_CREDITS` | Not enough credits to create campaign | 402 |

## Important Notes

<Info>
  **Campaign scheduling.** If no schedule is provided, the campaign will start immediately.
</Info>

<Info>
  **Rate limiting.** Campaigns are subject to rate limiting to prevent abuse.
</Info>

<Warning>
  **Phone number validation.** All phone numbers must be in E.164 format (e.g., +1234567890).
</Warning>

<Info>
  **Agent requirements.** The specified agent must be published and active to be used in campaigns.
</Info>

## Best Practices

1. **Test with small batches** - Start with a small number of recipients to test your campaign
2. **Respect business hours** - Use scheduling to avoid calling outside business hours
3. **Monitor campaign progress** - Track campaign status and adjust settings as needed
4. **Handle retries properly** - Configure appropriate retry attempts and delays
5. **Validate phone numbers** - Ensure all phone numbers are in the correct format
6. **Use metadata** - Include relevant metadata for personalization
7. **Monitor costs** - Campaign calls consume credits, so monitor usage
