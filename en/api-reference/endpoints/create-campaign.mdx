# Create Campaign

Create a new campaign with comprehensive configuration options including scheduling, retry logic, and do-not-call settings.

## Endpoint

```
POST /api/v1/campaigns
```

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Application JSON. Format: `application/json`
</ParamField>

## Request Body

<ParamField body="name" type="string" required>
  Campaign name (max 255 characters).
</ParamField>

<ParamField body="description" type="string" required={false}>
  Description of the campaign.
</ParamField>

<ParamField body="campaign_type" type="string" required>
  Type of campaign. Options: `sales`, `follow_up`, `reminder`, `custom`.
</ParamField>

<ParamField body="start_date" type="string" required>
  Campaign start date (YYYY-MM-DD).
</ParamField>

<ParamField body="days_of_week" type="array" required>
  Array of days when calls can be made (1=Monday, 7=Sunday).
</ParamField>

<ParamField body="earliest_call_time" type="string" required>
  Earliest time to make calls (HH:MM:SS).
</ParamField>

<ParamField body="latest_call_time" type="string" required>
  Latest time to make calls (HH:MM:SS).
</ParamField>

<ParamField body="timezone" type="string" required>
  Timezone for call scheduling (e.g., "America/New_York").
</ParamField>

<ParamField body="agent_id" type="string" required>
  UUID of the agent assigned to the campaign.
</ParamField>

<ParamField body="initial_call_delay" type="integer" required={false}>
  Initial delay before making calls in seconds (default: 0).
</ParamField>

<ParamField body="max_retries" type="integer" required={false}>
  Maximum number of retry attempts (default: 3).
</ParamField>

<ParamField body="retry_cooldown_hours" type="integer" required={false}>
  Hours to wait between retry attempts (default: 24).
</ParamField>

<ParamField body="enable_post_completion_cooldown" type="boolean" required={false}>
  Whether to enable post-completion cooldown (default: true).
</ParamField>

<ParamField body="post_completion_cooldown_hours" type="integer" required={false}>
  Hours to wait after campaign completion (default: 168).
</ParamField>

<ParamField body="success_cooldown_hours" type="integer" required={false}>
  Hours to wait after successful calls (default: 168).
</ParamField>

<ParamField body="voicemail_cooldown_hours" type="integer" required={false}>
  Hours to wait after voicemail calls (default: 24).
</ParamField>

<ParamField body="no_answer_cooldown_hours" type="integer" required={false}>
  Hours to wait after no-answer calls (default: 24).
</ParamField>

<ParamField body="busy_cooldown_hours" type="integer" required={false}>
  Hours to wait after busy calls (default: 24).
</ParamField>

<ParamField body="failed_cooldown_hours" type="integer" required={false}>
  Hours to wait after failed calls (default: 24).
</ParamField>

<ParamField body="do_not_call_enabled" type="boolean" required={false}>
  Whether to enable do-not-call list checking (default: true).
</ParamField>

<ParamField body="do_not_call_list_source" type="string" required={false}>
  Source of do-not-call list. Options: `environment`, `campaign` (default: "environment").
</ParamField>

<ParamField body="auto_add_to_dnc_enabled" type="boolean" required={false}>
  Whether to automatically add numbers to do-not-call list (default: true).
</ParamField>

<ParamField body="auto_dnc_trigger_statuses" type="array" required={false}>
  Call statuses that trigger auto-add to DNC. Options: `completed`, `voicemail` (default: ["completed", "voicemail"]).
</ParamField>

<ParamField body="auto_dnc_trigger_errors" type="array" required={false}>
  Call errors that trigger auto-add to DNC. Options: `invalid_number`, `disconnected` (default: ["invalid_number", "disconnected"]).
</ParamField>

## Important Notes

<Info>
  **Campaign status.** New campaigns are created with "draft" status and must be activated manually.
</Info>

<Info>
  **Timezone support.** Use valid timezone identifiers (e.g., "America/New_York", "Europe/London").
</Info>

<Info>
  **Days of week.** Use numbers 1-7 where 1=Monday and 7=Sunday.
</Info>

<Info>
  **Agent assignment.** The specified agent must exist and be active.
</Info>

<Warning>
  **Do-not-call compliance.** Ensure your campaign complies with local do-not-call regulations.
</Warning>

## Best Practices

1. **Start with draft status** - Create campaigns in draft mode and review before activating
2. **Set appropriate time windows** - Consider recipient time zones and business hours
3. **Configure retry logic** - Set reasonable retry limits and cooldown periods
4. **Enable DNC checking** - Always enable do-not-call list checking for compliance
5. **Test thoroughly** - Test campaign settings before going live
6. **Monitor performance** - Track campaign statistics and adjust settings as needed
