# Make a Call

Initiate an AI-powered call using a specific voice agent or campaign. This endpoint creates a new call and returns call details including a unique call ID.

## Endpoints

### Individual Agent Call

```
POST /api/v1/agents/{agent_id}/call
```

### Campaign Call

```
POST /api/v1/campaigns/{campaign_id}/call
```

## Path Parameters

<ParamField path="agent_id" type="string" required>
  The unique identifier of the voice agent that will handle the call. You can find this in your dashboard under Voice Agents.
</ParamField>

<ParamField path="campaign_id" type="string" required>
  The unique identifier of the campaign that will handle the call. You can find this in your dashboard under Campaigns.
</ParamField>

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be set to `application/json`
</ParamField>

## Request Body

### For Agent Calls

<ParamField body="to" type="string" required>
  The phone number to call. Must be in international format (e.g., +1234567890).
</ParamField>

### For Campaign Calls

<ParamField body="to" type="string" required>
  The phone number to call. Must be in international format (e.g., +1234567890).
</ParamField>

<ParamField body="payload" type="object" required>
  Custom data to pass to the campaign. This data will be available during the call for personalization.
</ParamField>

<ResponseExample>

```json
{
  "call_id": "call_789012",
  "agent_id": "agent_123456",
  "campaign_id": "campaign_789012",
  "to": "+1234567890",
  "status": "initiated",
  "created_at": "2024-01-15T10:30:00Z",
  "estimated_duration": 120,
  "payload": {
    "customer_name": "John Doe",
    "customer_id": "12345"
  }

```

</ResponseExample>

### Response Fields

<ResponseField name="call_id" type="string" required>
  Unique identifier for the call. Use this ID to track the call status and retrieve call details.
</ResponseField>

<ResponseField name="agent_id" type="string">
  The ID of the voice agent handling the call (for agent calls).
</ResponseField>

<ResponseField name="campaign_id" type="string">
  The ID of the campaign handling the call (for campaign calls).
</ResponseField>

<ResponseField name="to" type="string" required>
  The phone number that was called.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the call. Possible values:
  - `initiated` - Call has been created and is being processed
  - `ringing` - Phone is ringing
  - `in-progress` - Call is active and conversation is happening
  - `completed` - Call has finished successfully
  - `failed` - Call failed or was not answered
  - `busy` - Phone was busy
  - `no-answer` - Phone rang but was not answered
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp when the call was created.
</ResponseField>

<ResponseField name="estimated_duration" type="integer" required>
  Estimated duration of the call in seconds. This is based on the agent's or campaign's call flow configuration.
</ResponseField>

<ResponseField name="payload" type="object">
  The custom data passed with the call (for campaign calls). This data is available during the conversation for personalization.
</ResponseField>

## Error Codes

| Code | Description | HTTP Status |
|------|-------------|-------------|
| `INVALID_PHONE_NUMBER` | Phone number format is invalid | 400 |
| `MISSING_PHONE_NUMBER` | Phone number is required | 400 |
| `INVALID_TOKEN` | Authentication token is invalid or missing | 401 |
| `AGENT_NOT_FOUND` | Specified agent does not exist | 404 |
| `CAMPAIGN_NOT_FOUND` | Specified campaign does not exist | 404 |
| `AGENT_INACTIVE` | Agent is not active or configured | 400 |
| `CAMPAIGN_INACTIVE` | Campaign is not active or configured | 400 |
| `RATE_LIMIT_EXCEEDED` | Too many requests in a short time | 429 |
| `INSUFFICIENT_CREDITS` | Account has insufficient credits | 402 |

## Rate Limits

- **Standard Plan**: 10 calls per minute
- **Professional Plan**: 100 calls per minute
- **Enterprise Plan**: Custom limits

Rate limit headers are included in all responses:
- `X-RateLimit-Limit`: Maximum requests per window
- `X-RateLimit-Remaining`: Remaining requests in current window
- `X-RateLimit-Reset`: Time when the rate limit resets

## Call Status Flow

```
initiated → ringing → in-progress → completed
     ↓         ↓           ↓           ↓
   Call    Phone is    Agent is    Call has
  created   ringing   talking     finished
```

## Campaign Payload Usage

When making campaign calls, the `payload` object allows you to pass custom data that can be used during the conversation:

### Example Use Cases

- **Customer Information**: Pass customer name, ID, or account details
- **Order Information**: Include order number, items, or delivery details
- **Appointment Data**: Pass appointment time, location, or confirmation details
- **Survey Context**: Include survey type, questions, or previous responses

### Accessing Payload Data

The payload data is available during the call and can be referenced in your call flow for personalization:

```json
{
  "payload": {
    "customer_name": "John Doe",
    "order_number": "ORD-12345",
    "delivery_date": "2024-01-20"
  }
}
```

## Best Practices

1. **Always handle errors gracefully** - Check for error responses and implement retry logic
2. **Store call IDs** - Use the returned `call_id` to track call status
3. **Validate phone numbers** - Ensure phone numbers are in international format
4. **Monitor rate limits** - Check rate limit headers to avoid hitting limits
5. **Test with small numbers first** - Start with a few test calls before scaling
6. **Use meaningful payload data** - Pass relevant information for better call personalization
7. **Keep payload size reasonable** - Avoid very large payload objects
