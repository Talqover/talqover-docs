# Make External Call

Create external calls that use websocket transport instead of traditional telephony infrastructure. This endpoint allows you to integrate with external telephony providers while leveraging Talqover's AI voice agent capabilities.

## Overview

The Make External Call endpoint creates calls that bypass the traditional telephony flow and instead provide a websocket URL for external telephony providers to connect and handle the call audio streams.

## How It Works

When you call this endpoint, the system:
1. Creates an external call record
2. Returns a websocket URL for the external telephony provider
3. Sets the call status to "ready" for websocket connection
4. Provides transport information for external integration

## Endpoint

```
POST /api/v1/agents/{agent_id}/call
```

## Path Parameters

<ParamField path="agent_id" type="string" required>
  The unique identifier of the voice agent that will handle the call.
</ParamField>

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be set to `application/json`
</ParamField>

## Request Body

<ParamField body="sid" type="string" required>
  Session ID for the external call. This should be provided by your external telephony provider.
</ParamField>

<ParamField body="from" type="string" required>
  The phone number making the call. Must be in international format (e.g., +1234567890).
</ParamField>

<ParamField body="to" type="string" required>
  The phone number to call. Must be in international format (e.g., +1234567890).
</ParamField>

<ParamField body="direction" type="string" required>
  Call direction. Typically "outbound" for outgoing calls.
</ParamField>

<ParamField body="payload" type="array" required={false}>
  Custom data to pass to the call. This data will be available during the call for personalization.
</ParamField>

## Response Fields

<ResponseField name="id" type="string" required>
  Unique call identifier.
</ResponseField>

<ResponseField name="agent_id" type="string" required>
  The ID of the agent handling the call.
</ResponseField>

<ResponseField name="type" type="string" required>
  Call type, always "external.websocketCall" for external calls.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp when the call was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp when the call was last updated.
</ResponseField>

<ResponseField name="org_id" type="string" required>
  Organization/environment ID associated with the agent.
</ResponseField>

<ResponseField name="cost" type="number" required>
  Call cost. Always 0 for external calls.
</ResponseField>

<ResponseField name="status" type="string" required>
  Call status. "ready" indicates the call is ready for websocket connection.
</ResponseField>

<ResponseField name="transport" type="object" required>
  Transport configuration for the external call.
</ResponseField>

<ResponseField name="transport.provider" type="string" required>
  Transport provider type, always "external.websocket".
</ResponseField>

<ResponseField name="transport.websocketCallUrl" type="string" required>
  WebSocket URL for the external telephony provider to connect. The system automatically routes to the optimal regional endpoint based on your location.
</ResponseField>

## Regional Endpoints

Talqover provides regional WebSocket endpoints for optimal performance:

- **United States**: `wss://us-agent.talqover.app/{call_id}/transport`
- **Brazil**: `wss://br-agent.talqover.app/{call_id}/transport`
- **Europe**: `wss://eu-agent.talqover.app/{call_id}/transport`

The `websocketCallUrl` in the response will automatically point to the closest regional endpoint. For custom routing, you can manually construct the WebSocket URL using any of the regional endpoints.

## Important Notes

<Info>
  **External Call Only.** This endpoint is specifically designed for external telephony integration and bypasses traditional telephony infrastructure.
</Info>

<Info>
  **WebSocket Required.** After creating the call, you must connect to the provided websocket URL to handle the call audio streams.
</Info>

<Info>
  **Status Ready.** The call status "ready" indicates it's waiting for websocket connection from the external telephony provider.
</Info>

## Best Practices

1. **Validate Required Fields** - Ensure all required fields (sid, from, to, direction) are provided
2. **Use Unique SIDs** - Each call should have a unique session ID from your external provider
3. **Handle WebSocket Connection** - Connect to the websocket URL immediately after call creation
4. **Monitor Call Status** - Track the call status through the websocket connection
5. **Error Handling** - Implement proper error handling for validation and authentication failures
