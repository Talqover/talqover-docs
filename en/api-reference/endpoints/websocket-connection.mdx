# WebSocket Connection

Connect to the websocket transport endpoint to handle external telephony calls. This endpoint enables real-time audio streaming and call control for external telephony providers.

## Overview

The WebSocket Connection endpoint provides the real-time communication channel for external telephony providers to:
- Stream audio data to and from the AI voice agent
- Receive call control messages and status updates
- Handle call lifecycle events
- Manage audio quality and connection state

## Endpoint

The WebSocket transport endpoint is available in three regional locations:

**United States:**
```
WebSocket: wss://us-agent.talqover.app/{call_id}/transport?x-talq={token}
```

**Brazil:**
```
WebSocket: wss://br-agent.talqover.app/{call_id}/transport?x-talq={token}
```

**Europe:**
```
WebSocket: wss://eu-agent.talqover.app/{call_id}/transport?x-talq={token}
```

Choose the endpoint closest to your location for optimal performance and latency.

## Connection Parameters

<ParamField path="call_id" type="string" required>
  The call ID returned from the Make External Call endpoint.
</ParamField>

<ParamField query="x-talq" type="string" required>
  Authentication token for the websocket connection. This token is provided in the call creation response.
</ParamField>

## Connection Process

1. **Get Call ID** - Create an external call using the Make External Call endpoint
2. **Extract Token** - Get the authentication token from the call response
3. **Connect WebSocket** - Establish connection to the transport endpoint
4. **Handle Audio Streams** - Begin streaming audio data for the call

const wsUrl = usEndpoint; // or brEndpoint, euEndpoint
    
    this.ws = new WebSocket(wsUrl);

      console.log('Connected to Talqover WebSocket');
      this.isConnected = true;
      this.sendMessage({
        event: 'clear'
      });
    });

    this.ws.on('message', (data) => {
      this.handleMessage(data);
    });

    this.ws.on('close', () => {
      console.log('WebSocket connection closed');
      this.isConnected = false;
    });

    this.ws.on('error', (error) => {
      console.error('WebSocket error:', error);

            this.handleAgentAudio(message.media.payload);

          if (message.dtmf && message.dtmf.digit) {
            this.handleDTMF(message.dtmf.digit);
          }
          break;
        default:
          console.log('Unknown event type:', message.event);
      }
    } catch (error) {
      console.error('Error parsing message:', error);
    }
  }

  handleAgentAudio(audioData) {
    // Handle incoming base64 encoded audio from the AI agent
    // This is where you would process the audio for your telephony system
    console.log('Received agent audio:', audioData.length, 'characters (base64)');
  }

  handleDTMF(digit) {
    // Handle DTMF keypress from the agent
    console.log('DTMF keypress received:', digit);
  }

  disconnect() {
    if (this.ws) {
      this.ws.close();
    }
  }
}

## Event Protocol

The WebSocket connection uses a standardized event-based protocol for communication. All messages follow the same structure with an `event` field indicating the message type.

### Outgoing Messages (Client → Server)

| Event | Description | Payload |
|-------|-------------|---------|
| `media` | Sends audio data to the AI agent | `{ "event": "media", "media": { "payload": "base64_audio" } }` |
| `clear` | Clears interruption state | `{ "event": "clear" }` |

### Incoming Messages (Server → Client)

| Event | Description | Payload |
|-------|-------------|---------|
| `media` | Audio data from the AI agent | `{ "event": "media", "media": { "payload": "base64_audio" } }` |
| `dtmf` | DTMF keypress events | `{ "event": "dtmf", "dtmf": { "digit": "1" } }` |

### Message Structure

All messages follow this JSON structure:
```json
{
  "event": "event_type",
  "additional_fields": "values"
}
```

The `event` field is always required and determines how the message is processed.

## Audio Format

The WebSocket connection handles audio data in the following format:

- **Encoding**: Base64 encoded audio data
- **Format**: PCMU (μ-law) or PCMA (A-law) at 8kHz
- **Direction**: Bidirectional streaming
- **Sample Rate**: 8kHz
- **Channels**: Mono (1 channel)

## Connection States

### Connection Lifecycle

1. **Connecting** - WebSocket connection attempt
2. **Connected** - WebSocket connection established
3. **Ready** - Call is ready to start
4. **Active** - Call is in progress
5. **Ending** - Call is ending
6. **Disconnected** - WebSocket connection closed

## Error Handling

### Common WebSocket Errors

| Error | Description | Solution |
|-------|-------------|----------|
| `ECONNREFUSED` | Connection refused | Check if the endpoint is accessible |
| `ETIMEDOUT` | Connection timeout | Verify network connectivity |
| `ECONNRESET` | Connection reset | Reconnect with exponential backoff |
| `Unexpected server response` | Invalid response | Check authentication token |

### Reconnection Strategy

Implement a robust reconnection strategy:

```javascript
class WebSocketManager {
  constructor(url, options = {}) {
    this.url = url;
    this.options = {
      maxRetries: 5,
      retryDelay: 1000,
      ...options
    };
    this.retryCount = 0;
    this.connect();
  }

  connect() {
    try {
      this.ws = new WebSocket(this.url);
      this.setupEventHandlers();
    } catch (error) {
      this.handleConnectionError(error);
    }
  }

  setupEventHandlers() {
    this.ws.on('open', () => {
      console.log('Connected successfully');
      this.retryCount = 0;
    });

    this.ws.on('close', () => {
      console.log('Connection closed');
      this.scheduleReconnect();
    });

    this.ws.on('error', (error) => {
      console.error('WebSocket error:', error);
      this.handleConnectionError(error);
    });
  }

  scheduleReconnect() {
    if (this.retryCount < this.options.maxRetries) {
      const delay = this.options.retryDelay * Math.pow(2, this.retryCount);
      setTimeout(() => {
        this.retryCount++;
        console.log(`Reconnecting... Attempt ${this.retryCount}`);
        this.connect();
      }, delay);
    }
  }

  handleConnectionError(error) {
    console.error('Connection error:', error);
    this.scheduleReconnect();
  }
}
```

## Best Practices

1. **Immediate Connection** - Connect to the WebSocket immediately after call creation
2. **Heartbeat Monitoring** - Implement connection health monitoring
3. **Audio Buffering** - Buffer audio data to handle network latency
4. **Error Recovery** - Implement automatic reconnection with exponential backoff
5. **Resource Cleanup** - Properly close connections and clean up resources
