# Update Campaign

Update an existing campaign with partial or complete configuration changes. All fields are optional - only include the fields you want to update.

## Endpoint

```
PUT /api/v1/campaigns/{campaign}
```

## Path Parameters

<ParamField path="campaign" type="string" required>
  The unique identifier of the campaign. You can find this in your dashboard under Campaigns.
</ParamField>

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Format: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Application JSON. Format: `application/json`
</ParamField>

## Request Body

All fields are optional. Only include the fields you want to update.

<ParamField body="name" type="string" required={false}>
  Campaign name (max 255 characters).
</ParamField>

<ParamField body="description" type="string" required={false}>
  Description of the campaign.
</ParamField>

<ParamField body="campaign_type" type="string" required={false}>
  Type of campaign. Options: `sales`, `follow_up`, `reminder`, `custom`.
</ParamField>

<ParamField body="start_date" type="string" required={false}>
  Campaign start date (YYYY-MM-DD).
</ParamField>

<ParamField body="days_of_week" type="array" required={false}>
  Array of days when calls can be made (1=Monday, 7=Sunday).
</ParamField>

<ParamField body="earliest_call_time" type="string" required={false}>
  Earliest time to make calls (HH:MM:SS).
</ParamField>

<ParamField body="latest_call_time" type="string" required={false}>
  Latest time to make calls (HH:MM:SS).
</ParamField>

<ParamField body="timezone" type="string" required={false}>
  Timezone for call scheduling (e.g., "America/New_York").
</ParamField>

<ParamField body="agent_id" type="string" required={false}>
  UUID of the agent assigned to the campaign.
</ParamField>

<ParamField body="initial_call_delay" type="integer" required={false}>
  Initial delay before making calls in seconds.
</ParamField>

<ParamField body="max_retries" type="integer" required={false}>
  Maximum number of retry attempts.
</ParamField>

<ParamField body="retry_cooldown_hours" type="integer" required={false}>
  Hours to wait between retry attempts.
</ParamField>

<ParamField body="enable_post_completion_cooldown" type="boolean" required={false}>
  Whether to enable post-completion cooldown.
</ParamField>

<ParamField body="post_completion_cooldown_hours" type="integer" required={false}>
  Hours to wait after campaign completion.
</ParamField>

<ParamField body="success_cooldown_hours" type="integer" required={false}>
  Hours to wait after successful calls.
</ParamField>

<ParamField body="voicemail_cooldown_hours" type="integer" required={false}>
  Hours to wait after voicemail calls.
</ParamField>

<ParamField body="no_answer_cooldown_hours" type="integer" required={false}>
  Hours to wait after no-answer calls.
</ParamField>

<ParamField body="busy_cooldown_hours" type="integer" required={false}>
  Hours to wait after busy calls.
</ParamField>

<ParamField body="failed_cooldown_hours" type="integer" required={false}>
  Hours to wait after failed calls.
</ParamField>

<ParamField body="do_not_call_enabled" type="boolean" required={false}>
  Whether to enable do-not-call list checking.
</ParamField>

<ParamField body="do_not_call_list_source" type="string" required={false}>
  Source of do-not-call list. Options: `environment`, `campaign`.
</ParamField>

<ParamField body="auto_add_to_dnc_enabled" type="boolean" required={false}>
  Whether to automatically add numbers to do-not-call list.
</ParamField>

<ParamField body="auto_dnc_trigger_statuses" type="array" required={false}>
  Call statuses that trigger auto-add to DNC. Options: `completed`, `voicemail`.
</ParamField>

<ParamField body="auto_dnc_trigger_errors" type="array" required={false}>
  Call errors that trigger auto-add to DNC. Options: `invalid_number`, `disconnected`.
</ParamField>

## Example Requests

<RequestExample>

```bash

curl -X PUT "https://talqover.app/api/v1/campaigns/campaign-uuid-1" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Sales Campaign",
    "description": "Updated description for Q1 sales campaign"
  }'
```

```javascript
// Request 1: Update campaign name and description

    'Content-Type': 'application/json'
```

## Important Notes

<Info>
  **Partial updates.** Only include the fields you want to update - all fields are optional.
</Info>

<Info>
  **Status restrictions.** Some campaigns may have update restrictions based on their current status.
</Info>

<Info>
  **Validation rules.** All fields follow the same validation rules as campaign creation.
</Info>

<Warning>
  **Active campaigns.** Updating active campaigns may affect ongoing operations.
</Warning>

## Best Practices

1. **Update selectively** - Only update the fields that need to change
2. **Check status first** - Verify campaign status before attempting updates
3. **Validate changes** - Test updates on draft campaigns before applying to active ones
4. **Monitor effects** - Track how updates affect campaign performance
5. **Handle errors** - Implement proper error handling for validation and permission issues
