# Development Guide

This guide explains how to develop and maintain the Talkover documentation with support for multiple languages.

## Multi-Language Support

The documentation supports two languages:
- **English (EN)** - Default language
- **Brazilian Portuguese (PT-BR)** - Secondary language

### File Structure

```
talkover-docs/
├── en/                    # English content
│   ├── introduction.mdx
│   ├── essentials/
│   └── api-reference/
├── pt-br/                 # Portuguese content
│   ├── introduction.mdx
│   ├── essentials/
│   └── api-reference/
├── api-reference/         # Shared OpenAPI files
├── images/               # Shared images
└── docs.json            # Configuration
```

### Adding New Content

When adding new content, you must create versions for both languages:

1. **Create English version** in the `en/` directory
2. **Create Portuguese version** in the `pt-br/` directory
3. **Update navigation** in `docs.json` for both languages

### Translation Guidelines

- Maintain consistent terminology across languages
- Use proper Brazilian Portuguese (not European Portuguese)
- Keep technical terms consistent (API, endpoint, etc.)
- Ensure all links point to the correct language version

### Running Locally

```bash
# Install dependencies
npm install

# Start development server
mint dev
```

The documentation will be available at `http://localhost:3000` with language switching functionality.

### Building for Production

```bash
# Build the documentation
mint build
```

### Configuration

The `docs.json` file contains the main configuration:

- **Languages**: Define supported languages and their navigation
- **Navigation**: Structure for each language
- **Theme**: Colors, logo, and branding
- **Integrations**: Analytics, support widgets, etc.

### Best Practices

1. **Keep content synchronized** between languages
2. **Use consistent file naming** across language folders
3. **Test both languages** before deploying
4. **Update navigation** when adding new pages
5. **Use proper alt text** for images in both languages

### Contributing

When contributing to the documentation:

1. Create content for both languages
2. Follow the existing structure and style
3. Test your changes locally
4. Update this guide if needed
