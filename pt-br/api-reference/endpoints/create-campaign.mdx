# Criar Campanha

Criar uma nova campanha com configurações personalizadas incluindo agendamento, configurações de retry e regras de cooldown.

## Endpoint

```
POST /api/v1/campaigns
```

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Application JSON. Formato: `application/json`
</ParamField>

## Corpo da Requisição

<ParamField body="name" type="string" required>
  Nome da campanha (máximo 255 caracteres).
</ParamField>

<ParamField body="description" type="string" required={false}>
  Descrição da campanha.
</ParamField>

<ParamField body="campaign_type" type="string" required>
  Tipo da campanha. Opções: `sales`, `follow_up`, `reminder`, `custom`
</ParamField>

<ParamField body="start_date" type="string" required>
  Data de início da campanha (formato: YYYY-MM-DD).
</ParamField>

<ParamField body="days_of_week" type="array" required>
  Array de dias da semana (1=Segunda, 7=Domingo).
</ParamField>

<ParamField body="earliest_call_time" type="string" required>
  Horário mais cedo para fazer chamadas (formato: HH:MM:SS).
</ParamField>

<ParamField body="latest_call_time" type="string" required>
  Horário mais tarde para fazer chamadas (formato: HH:MM:SS).
</ParamField>

<ParamField body="timezone" type="string" required>
  Fuso horário para agendamento de chamadas.
</ParamField>

<ParamField body="agent_id" type="string" required>
  UUID do agente atribuído à campanha.
</ParamField>

<ParamField body="initial_call_delay" type="integer" required={false}>
  Atraso inicial da chamada em segundos (padrão: 0).
</ParamField>

<ParamField body="max_retries" type="integer" required={false}>
  Número máximo de tentativas (padrão: 3).
</ParamField>

<ParamField body="retry_cooldown_hours" type="integer" required={false}>
  Tempo de espera entre tentativas em horas (padrão: 24).
</ParamField>

<ParamField body="enable_post_completion_cooldown" type="boolean" required={false}>
  Habilitar cooldown pós-conclusão (padrão: false).
</ParamField>

<ParamField body="post_completion_cooldown_hours" type="integer" required={false}>
  Tempo de cooldown pós-conclusão em horas (padrão: 168).
</ParamField>

<ParamField body="success_cooldown_hours" type="integer" required={false}>
  Tempo de cooldown após sucesso em horas (padrão: 168).
</ParamField>

<ParamField body="voicemail_cooldown_hours" type="integer" required={false}>
  Tempo de cooldown após caixa postal em horas (padrão: 24).
</ParamField>

<ParamField body="no_answer_cooldown_hours" type="integer" required={false}>
  Tempo de cooldown após sem resposta em horas (padrão: 24).
</ParamField>

<ParamField body="busy_cooldown_hours" type="integer" required={false}>
  Tempo de cooldown após ocupado em horas (padrão: 24).
</ParamField>

<ParamField body="failed_cooldown_hours" type="integer" required={false}>
  Tempo de cooldown após falha em horas (padrão: 24).
</ParamField>

<ParamField body="do_not_call_enabled" type="boolean" required={false}>
  Habilitar lista de não chamar (padrão: false).
</ParamField>

<ParamField body="do_not_call_list_source" type="string" required={false}>
  Fonte da lista de não chamar. Opções: `environment`, `campaign`
</ParamField>

<ParamField body="auto_add_to_dnc_enabled" type="boolean" required={false}>
  Adicionar automaticamente à lista de não chamar (padrão: false).
</ParamField>

<ParamField body="auto_dnc_trigger_statuses" type="array" required={false}>
  Status que disparam adição automática à DNC. Opções: `completed`, `voicemail`
</ParamField>

<ParamField body="auto_dnc_trigger_errors" type="array" required={false}>
  Erros que disparam adição automática à DNC. Opções: `invalid_number`, `disconnected`
</ParamField>

## Exemplos de Requisição

<RequestExample>

```bash
# Requisição 1: Criar campanha de vendas básica
curl -X POST "https://talqover.app/api/v1/campaigns" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Campanha de Vendas Q1",
    "description": "Campanha de vendas outbound para Q1",
    "campaign_type": "sales",
    "start_date": "2024-01-01",
    "days_of_week": [1, 2, 3, 4, 5],
    "earliest_call_time": "09:00:00",
    "latest_call_time": "17:00:00",
    "timezone": "America/New_York",
    "agent_id": "agent-uuid-1",
    "max_retries": 3,
    "retry_cooldown_hours": 24
  }'
```

## Códigos de Erro

| Código | Descrição | Status HTTP |
|--------|-----------|-------------|
| `VALIDATION_ERROR` | Validação da requisição falhou | 422 |
| `AGENT_NOT_FOUND` | Agente especificado não existe | 404 |
| `INVALID_TOKEN` | Token de autenticação inválido ou ausente | 401 |
| `SERVER_ERROR` | Erro interno do servidor ocorreu | 500 |

## Notas Importantes

<Info>
  **Status inicial.** Campanhas são criadas com status "draft" por padrão.
</Info>

<Info>
  **Configurações de cooldown.** Configure tempos de espera apropriados para diferentes cenários.
</Info>

<Info>
  **Lista de não chamar.** Configure regras de DNC para compliance e eficiência.
</Info>

<Warning>
  **Fuso horário.** Certifique-se de usar o fuso horário correto para o público-alvo.
</Warning>

## Melhores Práticas

1. **Configure cooldowns** - Defina tempos de espera apropriados para diferentes cenários
2. **Use DNC** - Configure lista de não chamar para compliance
3. **Horários apropriados** - Defina horários de chamada que respeitem o público-alvo
4. **Teste configurações** - Teste as configurações antes de ativar a campanha
5. **Monitore desempenho** - Acompanhe o desempenho e ajuste conforme necessário

## Endpoints Relacionados

- **Listar Campanhas**: `GET /api/v1/campaigns`
- **Obter Campanha**: `GET /api/v1/campaigns/{campaign}`
- **Atualizar Campanha**: `PUT /api/v1/campaigns/{campaign}`
- **Atualizar Status da Campanha**: `PATCH /api/v1/campaigns/{campaign}/status`
- **Excluir Campanha**: `DELETE /api/v1/campaigns/{campaign}`
- **Fazer Chamada de Campanha**: `POST /api/v1/campaigns/{campaign}/call`
