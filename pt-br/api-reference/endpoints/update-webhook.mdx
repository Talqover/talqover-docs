---
title: "Atualizar Webhook"
api: "PUT /api/v1/agents/{agent_id}/webhooks/{webhook_id}"
description: "Atualizar uma configuração de webhook existente"
---

## Visão Geral

Atualiza uma configuração de webhook existente. Todos os campos são opcionais - apenas os campos fornecidos serão atualizados.

## Endpoint

<ParamField path="agent_id" type="string" required>
  UUID do Agente
</ParamField>

<ParamField path="webhook_id" type="string" required>
  UUID do Webhook
</ParamField>

## Requisição

<RequestField name="name" type="string">
  Nome legível para o webhook (máximo 255 caracteres)
</RequestField>

<RequestField name="webhook_url" type="string">
  URL onde os eventos do webhook serão enviados (máximo 500 caracteres, deve ser uma URL válida)
</RequestField>

<RequestField name="webhook_secret" type="string">
  Segredo opcional para verificação de assinatura HMAC-SHA256 (máximo 255 caracteres)
</RequestField>

<RequestField name="events" type="array">
  Array de tipos de eventos para se inscrever (mínimo 1 evento necessário). Cada evento deve estar na lista de eventos disponíveis.
</RequestField>

<RequestField name="timeout" type="integer">
  Timeout da requisição em segundos (5-120)
</RequestField>

<RequestField name="max_retries" type="integer">
  Número máximo de tentativas de nova tentativa (0-10)
</RequestField>

<RequestField name="enabled" type="boolean">
  Se o webhook deve estar ativo
</RequestField>

## Regras de Validação

- **name**: Opcional, string, máximo 255 caracteres
- **webhook_url**: Opcional, URL válida, máximo 500 caracteres
- **webhook_secret**: Opcional, string, máximo 255 caracteres
- **events**: Opcional, array com mínimo 1 evento, cada evento deve estar na lista de eventos disponíveis
- **timeout**: Opcional, inteiro entre 5-120 segundos
- **max_retries**: Opcional, inteiro entre 0-10
- **enabled**: Opcional, boolean

## Notas Importantes

- Todos os campos são opcionais - apenas os campos fornecidos serão atualizados
- Operações de webhook limpam automaticamente o cache do agente para efeito imediato
- Use endpoints HTTPS para URLs de webhook em produção
- Implemente verificação de assinatura adequada usando o segredo do webhook para segurança
