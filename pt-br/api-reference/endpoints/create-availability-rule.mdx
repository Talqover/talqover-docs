# Criar Regra de Disponibilidade

Adicionar uma regra de disponibilidade (janela recorrente ou data de exceção) a um calendário.

## Endpoint

```
POST /api/v1/calendars/{calendar_id}/availability-rules
```

## Parâmetros de Caminho

<ParamField path="calendar_id" type="string" required>UUID do calendário.</ParamField>

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>Deve ser `application/json`.</ParamField>

## Corpo da Requisição

<ParamField body="type" type="string" required>Tipo da regra. Opções: `recurring`, `exception`.</ParamField>

### Quando `type: "recurring"`

<ParamField body="days_of_week" type="array" required>Array de números de dias da semana (`1`=Seg ... `7`=Dom).</ParamField>
<ParamField body="start_time" type="string" required>Início da janela. Formato: `HH:MM`.</ParamField>
<ParamField body="end_time" type="string" required>Fim da janela. Formato: `HH:MM`.</ParamField>
<ParamField body="timezone" type="string">IANA timezone. Padrão é o fuso horário do calendário.</ParamField>

### Quando `type: "exception"`

<ParamField body="date" type="string" required>Data da exceção. Formato: `YYYY-MM-DD`.</ParamField>
<ParamField body="available" type="boolean" required>Indica se essa data está disponível (sobrescreve regras recorrentes) ou bloqueada.</ParamField>

## Exemplos

<RequestExample>

```bash
# Janela recorrente em dias úteis
curl -X POST "https://talkover.app/api/v1/calendars/calendar-uuid-1/availability-rules" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "recurring",
    "days_of_week": [1, 2, 3, 4, 5],
    "start_time": "09:00",
    "end_time": "17:00"
  }'
```

```bash
# Exceção de feriado (bloquear 25 de dezembro)
curl -X POST "https://talkover.app/api/v1/calendars/calendar-uuid-1/availability-rules" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{ "type": "exception", "date": "2024-12-25", "available": false }'
```

</RequestExample>

## Resposta

<ResponseExample>

```json
{
  "success": true,
  "data": {
    "id": "rule-uuid-1",
    "type": "recurring",
    "days_of_week": [1, 2, 3, 4, 5],
    "start_time": "09:00",
    "end_time": "17:00"
  }
}
```

</ResponseExample>
