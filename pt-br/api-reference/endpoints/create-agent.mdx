# Criar Agente

Criar um novo agente de voz com configurações personalizadas incluindo modelo de voz, base de conhecimento e configurações de comportamento.

## Endpoint

```
POST /api/v1/agents
```

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Application JSON. Formato: `application/json`
</ParamField>

## Corpo da Requisição

<ParamField body="name" type="string" required>
  Nome do agente (máximo 255 caracteres).
</ParamField>

<ParamField body="description" type="string" required={false}>
  Descrição do agente.
</ParamField>

<ParamField body="language" type="string" required>
  Idioma do agente. Opções: `en-US`, `pt-BR`, `es-ES`, `fr-FR`, `de-DE`, `it-IT`.
</ParamField>

<ParamField body="voice_template_id" type="string" required>
  ID do modelo de voz a ser usado pelo agente.
</ParamField>

<ParamField body="knowledge_base" type="string" required>
  Base de conhecimento do agente (texto ou URL).
</ParamField>

<ParamField body="interrupt_sensitivity" type="string" required={false}>
  Sensibilidade à interrupção. Opções: `low`, `medium`, `high` (padrão: "medium").
</ParamField>

<ParamField body="who_speaks_first" type="string" required={false}>
  Quem fala primeiro. Opções: `agent`, `human` (padrão: "agent").
</ParamField>

<ParamField body="initial_message" type="string" required={false}>
  Mensagem inicial do agente.
</ParamField>

<ParamField body="conversation_starters" type="array" required={false}>
  Array de iniciadores de conversa para o agente.
</ParamField>

## Exemplos de Requisição

<RequestExample>

```bash
# Requisição 1: Criar um agente básico de atendimento
curl -X POST "https://talqover.app/api/v1/agents" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Agente de Atendimento",
    "description": "Agente especializado em atendimento ao cliente",
    "language": "pt-BR",
    "voice_template_id": "voice-template-uuid-1",
    "knowledge_base": "Este é um agente de atendimento ao cliente especializado em resolver dúvidas e problemas dos clientes."
  }'
```

```javascript
// Requisição 1: Criar um agente básico de atendimento
const response = await fetch('https://talqover.app/api/v1/agents', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer talq_your_environment_token_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Agente de Atendimento',
    description: 'Agente especializado em atendimento ao cliente',
    language: 'pt-BR',
    voice_template_id: 'voice-template-uuid-1',
    knowledge_base: 'Este é um agente de atendimento ao cliente especializado em resolver dúvidas e problemas dos clientes.'
  })
});

const result = await response.json();
console.log(result);
```

</RequestExample>

  "success": true,

</ResponseExample>

    <ResponseField name="voice_template_id" type="string" required>

    </ResponseField>
  </Expandable>

## Códigos de Erro

| Código | Descrição | Status HTTP |
|--------|-----------|-------------|
| `VALIDATION_ERROR` | Validação da requisição falhou | 422 |
| `INVALID_TOKEN` | Token de autenticação inválido ou ausente | 401 |
| `VOICE_TEMPLATE_NOT_FOUND` | Modelo de voz especificado não existe | 404 |
| `SERVER_ERROR` | Erro interno do servidor ocorreu | 500 |

## Notas Importantes

<Info>
  **Status inicial.** Novos agentes são criados com status "draft" e devem ser publicados manualmente.
</Info>

<Info>
  **Modelo de voz.** O modelo de voz especificado deve existir e estar ativo.
</Info>

<Info>
  **Base de conhecimento.** Forneça uma base de conhecimento detalhada para melhor desempenho do agente.
</Info>

<Warning>
  **Limites de caracteres.** A base de conhecimento tem limites de caracteres que devem ser respeitados.
</Warning>

## Melhores Práticas

1. **Base de conhecimento detalhada** - Forneça informações abrangentes para melhor desempenho
2. **Teste antes de publicar** - Sempre teste agentes em modo rascunho antes de publicar
3. **Escolha o modelo de voz adequado** - Selecione um modelo de voz apropriado para o idioma
4. **Configure comportamento** - Ajuste sensibilidade à interrupção e quem fala primeiro conforme necessário
5. **Mensagem inicial clara** - Forneça uma mensagem inicial clara e profissional

## Endpoints Relacionados

- **Listar Agentes**: `GET /api/v1/agents`
- **Obter Agente**: `GET /api/v1/agents/{agent}`
- **Atualizar Conhecimento do Agente**: `PUT /api/v1/agents/{agent}/knowledge`
- **Publicar Agente**: `POST /api/v1/agents/{agent}/publish`
