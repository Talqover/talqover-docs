# Fazer Chamada Externa

Criar chamadas externas que usam transporte websocket em vez de infraestrutura de telefonia tradicional. Este endpoint permite que você integre com provedores de telefonia externos enquanto aproveita as capacidades de agentes de voz com IA do Talqover.

## Visão Geral

O endpoint Fazer Chamada Externa cria chamadas que contornam o fluxo de telefonia tradicional e, em vez disso, fornecem uma URL websocket para provedores de telefonia externos se conectarem e gerenciarem os streams de áudio da chamada.

## Chamadas Individuais do Agente vs Chamadas de Campanha

### **Chamadas Individuais do Agente** - Interações Imediatas e de Alto Impacto
As chamadas individuais do agente são projetadas para **engajamento instantâneo e personalizado com clientes** que impulsiona resultados imediatos:

- **Execução Instantânea**: As chamadas são feitas imediatamente sem esperar pelo agendamento da campanha ou horários comerciais
- **Contexto em Tempo Real**: Perfeito para acompanhamentos imediatos após envios de formulários, interações no site ou ações do cliente
- **Conversões de Alto Valor**: Ideal para leads de alta prioridade e necessidades urgentes do cliente
- **Parâmetros Flexíveis**: Personalize os parâmetros da chamada em tempo real para situações específicas
- **Controle Direto**: Controle imediato sobre o tempo, seleção do agente e execução
- **Toque Pessoal**: Interação semelhante à humana que parece imediata e responsiva

**Melhor para**: 
- **Acompanhamentos de formulários** - Formulários de contato, solicitações de cotação, inscrições para demo
- **Atendimento imediato ao cliente** - Problemas urgentes, suporte de alta prioridade
- **Conversões de vendas rápidas** - Acompanhamentos pós-demo, recuperação de carrinho abandonado
- **Engajamento em tempo real** - Escalações de chat ao vivo, chamadas de visitantes do site
- **Interações de alto valor** - Clientes VIP, prospects empresariais

### **Chamadas de Campanha** - Alcance Estratégico e Automatizado
As chamadas de campanha são projetadas para **engajamento sistemático e em larga escala com clientes** com automação avançada:

- **Configurações Específicas da Campanha**: Herda a configuração da campanha (lógica de nova tentativa, períodos de cooldown, horários de chamada)
- **Agendamento Avançado**: Respeita horários comerciais, configurações de fuso horário e restrições de dia
- **Processamento em Lote**: Gerencie milhares de chamadas com enfileiramento inteligente
- **Análise de Performance**: Rastreamento e relatórios abrangentes
- **Recursos de Conformidade**: Gerenciamento de lista de não ligar e ferramentas regulatórias

**Melhor para**: Nutrição de leads, lembretes de compromissos, pesquisa de mercado, pesquisas de satisfação do cliente e alcance sistemático.

## Como Funciona

Quando você chama este endpoint, o sistema:
1. Cria um registro de chamada externa
2. Retorna uma URL websocket para o provedor de telefonia externo
3. Define o status da chamada como "ready" para conexão websocket
4. Fornece informações de transporte para integração externa

## Endpoint

```
POST /api/v1/agents/{agent_id}/call
```

## Parâmetros de Caminho

<ParamField path="agent_id" type="string" required>
  O identificador único do agente de voz que irá gerenciar a chamada.
</ParamField>

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Deve ser definido como `application/json`
</ParamField>

## Corpo da Requisição

<ParamField body="sid" type="string" required>
  ID da sessão para a chamada externa. Este deve ser fornecido pelo seu provedor de telefonia externo.
</ParamField>

<ParamField body="from" type="string" required>
  O número de telefone fazendo a chamada. Deve estar no formato internacional (ex: +1234567890).
</ParamField>

<ParamField body="to" type="string" required>
  O número de telefone para chamar. Deve estar no formato internacional (ex: +1234567890).
</ParamField>

<ParamField body="direction" type="string" required>
  Direção da chamada. Tipicamente "outbound" para chamadas de saída.
</ParamField>

<ParamField body="payload" type="array" required={false}>
  Dados personalizados para passar para a chamada. Esses dados estarão disponíveis durante a chamada para personalização.
</ParamField>

## Campos de Resposta

<ResponseField name="id" type="string" required>
  Identificador único da chamada.
</ResponseField>

<ResponseField name="agent_id" type="string" required>
  O ID do agente gerenciando a chamada.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipo da chamada, sempre "external.websocketCall" para chamadas externas.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  Timestamp ISO 8601 quando a chamada foi criada.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  Timestamp ISO 8601 quando a chamada foi atualizada pela última vez.
</ResponseField>

<ResponseField name="org_id" type="string" required>
  ID da organização/ambiente associado ao agente.
</ResponseField>

<ResponseField name="cost" type="number" required>
  Custo da chamada. Sempre 0 para chamadas externas.
</ResponseField>

<ResponseField name="status" type="string" required>
  Status da chamada. "ready" indica que a chamada está pronta para conexão websocket.
</ResponseField>

<ResponseField name="transport" type="object" required>
  Configuração de transporte para a chamada externa.
</ResponseField>

<ResponseField name="transport.provider" type="string" required>
  Tipo do provedor de transporte, sempre "external.websocket".
</ResponseField>

<ResponseField name="transport.websocketCallUrl" type="string" required>
  URL WebSocket para o provedor de telefonia externo se conectar. O sistema roteia automaticamente para o endpoint regional ideal com base na sua localização.
</ResponseField>

## Endpoints Regionais

O Talqover fornece endpoints WebSocket regionais para performance ideal:

- **Estados Unidos**: `wss://us-agent.talqover.app/{call_id}/transport`
- **Brasil**: `wss://br-agent.talqover.app/{call_id}/transport`
- **Europa**: `wss://eu-agent.talqover.app/{call_id}/transport`

A `websocketCallUrl` na resposta apontará automaticamente para o endpoint regional mais próximo. Para roteamento personalizado, você pode construir manualmente a URL WebSocket usando qualquer um dos endpoints regionais.

## Notas Importantes

<Info>
  **Chamada Externa Apenas.** Este endpoint é especificamente projetado para integração de telefonia externa e contorna a infraestrutura de telefonia tradicional.
</Info>

<Info>
  **WebSocket Obrigatório.** Após criar a chamada, você deve se conectar à URL websocket fornecida para gerenciar os streams de áudio da chamada.
</Info>

<Info>
  **Status Pronto.** O status da chamada "ready" indica que está aguardando conexão websocket do provedor de telefonia externo.
</Info>

## Melhores Práticas

1. **Validar Campos Obrigatórios** - Certifique-se de que todos os campos obrigatórios (sid, from, to, direction) sejam fornecidos
2. **Usar SIDs Únicos** - Cada chamada deve ter um ID de sessão único do seu provedor externo
3. **Gerenciar Conexão WebSocket** - Conecte-se à URL websocket imediatamente após a criação da chamada
4. **Monitorar Status da Chamada** - Acompanhe o status da chamada através da conexão websocket
5. **Tratamento de Erros** - Implemente tratamento adequado de erros para falhas de validação e autenticação
