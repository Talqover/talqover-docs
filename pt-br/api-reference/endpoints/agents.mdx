# API de Agentes de Voz

Este documento fornece documentação abrangente para os endpoints da API de Agentes de Voz, incluindo exemplos de requisição/resposta e regras de validação.

## URL Base

```
/api/v1/agents
```

## Autenticação

Todos os endpoints requerem autenticação via token de ambiente através do middleware `EnvironmentTokenAuth`.

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

## Operações Principais

#### Parâmetros de Consulta

<ParamField query="status" type="string" required={false}>
  Filtrar por status do agente. Opções: `draft`, `published`
</ParamField>

<ParamField query="direction" type="string" required={false}>
  Filtrar por direção da chamada. Opções: `inbound`, `outbound`
</ParamField>

</ParamField>

<ParamField query="per_page" type="integer" required={false}>

</ParamField>

### Criar Agente

<RequestExample>

```bash
curl -X POST "https://talqover.app/api/v1/agents" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "label": "Agente de Suporte",
    "description": "Gerencia consultas e solicitações de suporte ao cliente",
    "name": "SupportBot",
    "language": "pt-BR",
    "initial_message": "Olá, como posso ajudá-lo hoje?",
    "interrupt_sensitivity": "medium",
    "endpointing_sensitivity": "auto",
    "ivr_navigation_mode": "off",
    "conversation_speed": 1.0,

    "llm_temperature": 0.7,
    "enable_recording": true,
    "direction": "inbound",
```

#### Parâmetros do Corpo da Requisição

<ParamField body="label" type="string" required>
  Rótulo do agente (máx 255 caracteres)
</ParamField>

<ParamField body="description" type="string" required={false}>
  Descrição do agente
</ParamField>

<ParamField body="name" type="string" required>
  Nome do agente (máx 255 caracteres)
</ParamField>

<ParamField body="language" type="string" required>
  Idioma do agente. Opções: `pt-BR`, `en-US`, `es-ES`, etc.
</ParamField>

<ParamField body="initial_message" type="string" required={false}>
  Mensagem inicial que o agente dirá
</ParamField>

<ParamField body="interrupt_sensitivity" type="string" required>
  Nível de sensibilidade à interrupção. Opções: `low`, `medium`, `high`
</ParamField>

<ParamField body="endpointing_sensitivity" type="string" required>
  Sensibilidade de endpointing (máx 255 caracteres)
</ParamField>

<ParamField body="ivr_navigation_mode" type="string" required>
  Modo de navegação IVR (máx 255 caracteres)
</ParamField>

<ParamField body="conversation_speed" type="number" required>
  Velocidade da conversa (mín: 0.1, máx: 3.0)
</ParamField>

<ParamField body="initial_message_delay" type="integer" required>
  Atraso da mensagem inicial em segundos (mín: 0, máx: 60)
</ParamField>

<ParamField body="ask_if_human_present_on_idle" type="boolean" required>
  Se deve perguntar se há humano presente quando ocioso
</ParamField>

<ParamField body="idle_time_seconds" type="integer" required>
  Tempo ocioso em segundos (mín: 1, máx: 300)
</ParamField>

<ParamField body="llm_temperature" type="number" required>
  Temperatura do LLM (mín: 0, máx: 2)
</ParamField>

<ParamField body="enable_recording" type="boolean" required>
  Se deve habilitar gravação de chamadas
</ParamField>

<ParamField body="direction" type="string" required>
  Direção da chamada. Opções: `inbound`, `outbound`
</ParamField>

<ParamField body="who_speaks_first" type="string" required>
  Quem fala primeiro. Opções: `agent`, `user`
</ParamField>

## Referência de Enums

### Status do Agente
- `draft`: Agente está em modo rascunho
- `published`: Agente está publicado e ativo

### Direção do Agente
- `inbound`: Agente recebe chamadas recebidas
- `outbound`: Agente faz chamadas de saída

### Idioma do Agente
- `pt-BR`: Português (Brasil)
- `en-US`: Inglês (Estados Unidos)
- `es-ES`: Espanhol (Espanha)
- E outros idiomas suportados

### Sensibilidade à Interrupção
- `low`: Baixa sensibilidade à interrupção
- `medium`: Sensibilidade média à interrupção
- `high`: Alta sensibilidade à interrupção

### Quem Fala Primeiro
- `agent`: Agente fala primeiro
- `user`: Usuário fala primeiro

### Tipos de Ação
- `webhook`: Ação de webhook
- `transfer`: Ação de transferência de chamada
- `hold`: Ação de retenção de chamada
- E outros tipos de ação

### Status Ativo
- `active`: Item está ativo
- `inactive`: Item está inativo
