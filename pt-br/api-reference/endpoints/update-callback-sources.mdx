# Atualizar Origens de Callback

Substitui a lista de agentes de origem associados a este agente de callback. A lista fornecida substitui completamente a existente.

## Endpoint

```
PUT /api/v1/agents/{agent_id}/callback
```

## Parâmetros de Caminho

<ParamField path="agent_id" type="string" required>UUID do agente de callback.</ParamField>

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Deve ser definido como `application/json`
</ParamField>

## Corpo da Requisição

<ParamField body="source_agent_ids" type="array" required>
  Nova lista completa de UUIDs de agentes outbound que devem rotear callbacks para este agente. Deve conter pelo menos uma entrada.
</ParamField>

## Exemplos

<RequestExample>

```bash
curl -X PUT "https://talkover.app/api/v1/agents/agent-uuid-1/callback" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "source_agent_ids": ["agent-uuid-2"]
  }'
```

</RequestExample>

## Resposta

<ResponseExample>

```json
{
  "success": true,
  "data": {
    "id": "agent-uuid-1",
    "callback_config": {
      "source_agent_ids": ["agent-uuid-2"]
    }
  }
}
```

</ResponseExample>

## Observações

- Esta é uma operação de **substituição**, não de adição. Para remover todas as origens mas manter o papel de callback, envie um array vazio — porém note que um array vazio falha na validação; use [Remover Papel de Callback](/api-reference/endpoints/remove-callback-role) em vez disso.
- Para adicionar ou remover origens individuais, busque a lista atual com [Obter Agente de Callback](/api-reference/endpoints/get-callback-agent), modifique-a e então envie a lista atualizada completa aqui.
