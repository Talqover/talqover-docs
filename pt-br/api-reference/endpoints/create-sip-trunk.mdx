# Criar SIP Trunk

Registra um SIP Trunk personalizado para o seu ambiente. Após criado, é necessário verificar o trunk via [Verificar SIP Trunk](/api-reference/endpoints/verify-sip-trunk) antes que ele possa ser utilizado.

<Info>
  Este recurso está disponível apenas em planos onde `allow_custom_sip_trunks` está habilitado.
</Info>

## Endpoint

```
POST /api/v1/sip-trunks
```

## Cabeçalhos da requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Deve ser definido como `application/json`
</ParamField>

## Corpo da requisição

<ParamField body="name" type="string" required>Nome legível do trunk. Máximo de 255 caracteres.</ParamField>
<ParamField body="provider_name" type="string" required>Identificador da sua operadora ou provedor SIP.</ParamField>
<ParamField body="domain" type="string" required>Domínio SIP ou host do proxy (ex: `sip.carrier-x.com`).</ParamField>
<ParamField body="username" type="string" required>Nome de usuário para autenticação SIP.</ParamField>
<ParamField body="password" type="string" required>Senha para autenticação SIP. Armazenada de forma criptografada.</ParamField>
<ParamField body="port" type="integer">Porta SIP. Faixa: `1`–`65535`. O padrão depende do transporte (`5060` UDP/TCP, `5061` TLS).</ParamField>
<ParamField body="transport" type="string">Protocolo de transporte. Opções: `udp`, `tcp`, `tls`. Padrão: `tls`.</ParamField>
<ParamField body="outbound_proxy" type="string">Host do proxy outbound opcional.</ParamField>
<ParamField body="auth_realm" type="string">Realm de autenticação SIP opcional, caso a sua operadora exija.</ParamField>
<ParamField body="is_active" type="boolean">Indica se o trunk está ativo. Padrão: `false` (você deve verificar antes de ativar).</ParamField>

## Exemplos

<RequestExample>

```bash
curl -X POST "https://app.talkover.ai/api/v1/sip-trunks" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Primary Carrier — US",
    "provider_name": "carrier-x",
    "domain": "sip.carrier-x.com",
    "username": "user_42",
    "password": "secret-password",
    "port": 5061,
    "transport": "tls"
  }'
```

</RequestExample>

## Resposta

<ResponseExample>

```json
{
  "success": true,
  "data": {
    "id": "trunk-uuid-1",
    "name": "Primary Carrier — US",
    "provider_name": "carrier-x",
    "domain": "sip.carrier-x.com",
    "transport": "tls",
    "port": 5061,
    "is_active": false,
    "is_verified": false,
    "created_at": "2024-01-15T14:00:00Z"
  }
}
```

</ResponseExample>

## Observações

- O `password` nunca é retornado nas respostas após a criação.
- Execute [Verificar SIP Trunk](/api-reference/endpoints/verify-sip-trunk) para testar a conectividade antes de ativar.
