---

## Visão Geral

Cria uma nova configuração de webhook para um agente. Webhooks permitem receber notificações em tempo real quando eventos específicos ocorrem durante chamadas.

## Endpoint

<ParamField path="agent_id" type="string" required>
  UUID do Agente
</ParamField>

## Requisição

<RequestField name="name" type="string" required>
  Nome legível para o webhook (máximo 255 caracteres)
</RequestField>

<RequestField name="webhook_url" type="string" required>
  URL onde os eventos do webhook serão enviados (máximo 500 caracteres, deve ser uma URL válida)
</RequestField>

<RequestField name="webhook_secret" type="string">
  Segredo opcional para verificação de assinatura HMAC-SHA256 (máximo 255 caracteres)
</RequestField>

<RequestField name="events" type="array" required>
  Array de tipos de eventos para se inscrever (mínimo 1 evento necessário). Cada evento deve estar na lista de eventos disponíveis.
</RequestField>

<RequestField name="timeout" type="integer">
  Timeout da requisição em segundos (5-120, padrão: 30)
</RequestField>

<RequestField name="max_retries" type="integer">
  Número máximo de tentativas de nova tentativa (0-10, padrão: 3)
</RequestField>

<RequestField name="enabled" type="boolean">
  Se o webhook deve estar ativo (padrão: true)
</RequestField>

## Regras de Validação

- **name**: Obrigatório, string, máximo 255 caracteres
- **webhook_url**: Obrigatório, URL válida, máximo 500 caracteres
- **webhook_secret**: Opcional, string, máximo 255 caracteres
- **events**: Obrigatório, array com mínimo 1 evento, cada evento deve estar na lista de eventos disponíveis
- **timeout**: Opcional, inteiro entre 5-120 segundos (padrão: 30)
- **max_retries**: Opcional, inteiro entre 0-10 (padrão: 3)
- **enabled**: Opcional, boolean (padrão: true)

## Notas Importantes

- Operações de webhook limpam automaticamente o cache do agente para efeito imediato
- Use endpoints HTTPS para URLs de webhook em produção
- Implemente verificação de assinatura adequada usando o segredo do webhook para segurança
- Teste seu endpoint de webhook antes de ir ao ar usando o endpoint de teste
