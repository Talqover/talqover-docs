# Atualizar Configuração SFTP da Campanha

Atualiza a configuração SFTP de uma campanha. Habilitar (`is_enabled: true`) requer um teste de conexão bem-sucedido registrado — caso contrário, a requisição falha com 422.

## Endpoint

```
PUT /api/v1/campaigns/{campaign_id}/sftp
```

## Parâmetros de Caminho

<ParamField path="campaign_id" type="string" required>UUID da campanha.</ParamField>

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Deve ser definido como `application/json`
</ParamField>

## Corpo da Requisição

Mesmos campos que [Criar Configuração SFTP da Campanha](/api-reference/endpoints/create-campaign-sftp), todos opcionais, mais:

<ParamField body="is_enabled" type="boolean">
  Alterna a integração. Definir como `true` requer um resultado de teste bem-sucedido.
</ParamField>

## Exemplos

<RequestExample>

```bash
curl -X PUT "https://app.talkover.ai/api/v1/campaigns/campaign-uuid/sftp" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "import_schedule": "hourly",
    "is_enabled": true
  }'
```

</RequestExample>

## Resposta

<ResponseExample>

```json
{
  "success": true,
  "data": {
    "id": "sftp-config-uuid-1",
    "import_schedule": "hourly",
    "is_enabled": true,
    "updated_at": "2024-01-15T16:30:00Z"
  }
}
```

</ResponseExample>

## Respostas de Erro

### 422 — Não é Possível Habilitar

<ResponseExample>

```json
{
  "success": false,
  "message": "Run a successful test connection before enabling SFTP",
  "code": "SFTP_TEST_REQUIRED"
}
```

</ResponseExample>
