# Criar/Atualizar Ação

Criar ou atualizar uma ação para um agente específico. Este endpoint suporta diferentes tipos de ações: webhooks, transferências e pausas.

## Endpoint

```
POST /api/v1/agents/{agent}/actions
```

## Parâmetros de Caminho

<ParamField path="agent" type="string" required>
  O identificador único do agente. Você pode encontrá-lo na lista de agentes.
</ParamField>

## Cabeçalhos da Requisição

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticação. Formato: `Bearer talq_your_environment_token_here`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Application JSON. Formato: `application/json`
</ParamField>

## Corpo da Requisição

<ParamField body="type" type="string" required>
  Tipo da ação. Opções: `webhook`, `transfer`, `hold`
</ParamField>

<ParamField body="name" type="string" required>
  Nome da ação (máximo 255 caracteres).
</ParamField>

<ParamField body="description" type="string" required={false}>
  Descrição da ação.
</ParamField>

<ParamField body="config" type="object" required>
  Configuração específica da ação baseada no tipo.

  <Expandable title="Configuração de Webhook">
    <ParamField body="config.url" type="string" required>
      URL do webhook.
    </ParamField>
    <ParamField body="config.method" type="string" required>
      Método HTTP. Opções: `GET`, `POST`, `PUT`, `DELETE`
    </ParamField>
    <ParamField body="config.headers" type="object" required={false}>
      Cabeçalhos HTTP para a requisição.
    </ParamField>
  </Expandable>

  <Expandable title="Configuração de Transferência">
    <ParamField body="config.phone_number" type="string" required>
      Número de telefone para transferência em formato E.164.
    </ParamField>
    <ParamField body="config.transfer_message" type="string" required={false}>
      Mensagem de transferência.
    </ParamField>
  </Expandable>

  <Expandable title="Configuração de Pausa">
    <ParamField body="config.duration" type="integer" required>
      Duração da pausa em segundos.
    </ParamField>
    <ParamField body="config.message" type="string" required={false}>
      Mensagem durante a pausa.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="is_active" type="boolean" required={false}>
  Se a ação deve estar ativa (padrão: true).
</ParamField>

## Exemplos de Requisição

<RequestExample>

```bash
# Requisição 1: Criar webhook
curl -X POST "https://talqover.app/api/v1/agents/agent-uuid-1/actions" \
  -H "Authorization: Bearer talq_your_environment_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "webhook",
    "name": "Webhook de Notificação",
    "description": "Envia notificação para sistema externo",
    "config": {
      "url": "https://api.example.com/webhook",
      "method": "POST",
      "headers": {
        "Content-Type": "application/json",
        "Authorization": "Bearer webhook-token"
      }
    },
    "is_active": true
  }'
```

```javascript
// Requisição 1: Criar webhook
const response = await fetch('https://talqover.app/api/v1/agents/agent-uuid-1/actions', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer talq_your_environment_token_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    type: 'webhook',
    name: 'Webhook de Notificação',
    description: 'Envia notificação para sistema externo',
    config: {
      url: 'https://api.example.com/webhook',
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer webhook-token'
      }
    },
    is_active: true
  })
});

const result = await response.json();
console.log(result);
```

</RequestExample>

## Códigos de Erro

| Código | Descrição | Status HTTP |
|--------|-----------|-------------|
| `AGENT_NOT_FOUND` | Agente especificado não existe | 404 |
| `VALIDATION_ERROR` | Validação da requisição falhou | 422 |
| `INVALID_TOKEN` | Token de autenticação inválido ou ausente | 401 |
| `SERVER_ERROR` | Erro interno do servidor ocorreu | 500 |

## Notas Importantes

<Info>
  **Tipos de ação.** Suporta webhooks, transferências e pausas com configurações específicas.
</Info>

<Info>
  **Configurações obrigatórias.** Cada tipo de ação requer configurações específicas.
</Info>

<Info>
  **Status ativo.** Ações podem ser criadas como ativas ou inativas.
</Info>

<Warning>
  **URLs válidas.** Para webhooks, certifique-se de que a URL é válida e acessível.
</Warning>

## Melhores Práticas

1. **Teste webhooks** - Sempre teste webhooks antes de usar em produção
2. **URLs seguras** - Use HTTPS para URLs de webhook
3. **Configurações válidas** - Verifique se todas as configurações obrigatórias estão presentes
4. **Nomes descritivos** - Use nomes claros e descritivos para as ações
5. **Documentação** - Mantenha descrições detalhadas para cada ação

- **Listar Agentes**: `GET /api/v1/agents`
